# PoC1: 离线批处理 + 快速检索验证系统

## 🎯 目标与定位

### 核心目标
验证V3设计中的核心识别功能质量，采用**最简单可行**的架构，专注于：
1. **验证识别准确性** - 不追求性能，先证明识别能力
2. **降低系统复杂度** - 避免过早优化
3. **快速验证反馈** - 2周内可交付可测试的原型

### 范围界定
**包含：**
- ✅ 离线批量图片导入和处理
- ✅ 基础物体检测（使用RTMDet，52.8% mAP精度）
- ✅ OCR文本提取功能（PaddleOCR）
- ✅ 简单的元数据搜索
- ✅ 基础Web UI展示和搜索

**不包含：**
- ❌ 实时处理和性能优化
- ❌ Few-shot学习功能
- ❌ 品牌/型号细分识别
- ❌ 向量搜索和复杂检索
- ❌ 用户认证和多租户

## 📊 成功标准

1. **功能验证**
   - 能够批量处理100-1000张图片
   - 大类识别准确率达到70-80%
   - OCR提取准确率达到85%以上
   - 搜索功能可用且响应合理

2. **用户体验**
   - 批处理完成后能快速浏览结果
   - 搜索结果相关性高
   - UI简洁直观，无需培训即可使用

3. **技术验证**
   - 架构简单，易于理解和修改
   - 依赖最少，部署简单
   - 代码清晰，易于后续迭代

## 🏗 简化架构

```
用户 → Web UI → FastAPI → 批处理器 → 数据库
                  ↓          ↓
              搜索API    识别引擎
                         (RTMDet/CLIP + OCR)
```

## 📅 时间规划

- **第1-3天**: 搭建基础框架，实现批处理流程
- **第4-7天**: 集成识别引擎，完成核心功能
- **第8-10天**: 开发简单UI，实现搜索功能
- **第11-14天**: 测试优化，准备演示
